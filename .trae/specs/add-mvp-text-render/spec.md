# clishot MVP（V1 Render-only）Spec

## Why
实验报告需要“终端截图”，但真实截图受窗口尺寸、不可见终端（sandbox）、GUI 依赖等限制。clishot V1 通过“纯文本 → 统一排版 → 图片输出”提供稳定、可复现、可插入报告的交付物。

## What Changes
- 新增 CLI：从 `stdin` 或文本文件读取内容，清洗与换行后渲染为一张或多张图片（PNG/JPG）。
- 新增分页：长内容自动拆分页并按顺序输出多张图片。
- 新增主题：至少提供 1 个默认主题；可选 `--theme` 切换（最少 `terminal` / `paper` 两种）。
- 新增一致性保障：等宽字体渲染；对常见控制字符与 `\r` 覆盖行给出确定性处理策略。

## Impact
- Affected specs: CLI 输入输出、文本清洗、换行与分页、渲染主题与字体一致性、测试（单测 + E2E）。
- Affected code: 新增/修改命令行入口、渲染模块、主题与字体资源、测试用例（路径以实现时的工程结构为准）。

## ADDED Requirements
### Requirement: Render 命令（纯文本输入 → 图片输出）
系统 SHALL 提供 CLI 渲染能力，将纯文本内容渲染为图片文件，输出结果在同一输入与参数下保持确定性。

#### Scenario: 从 stdin 渲染单页
- **WHEN** 用户通过管道/重定向向 CLI 提供文本输入
- **AND** 指定 `--out` 输出路径为单个文件名（如 `out.png`）
- **THEN** 系统生成图片文件
- **AND** 图片内容包含输入文本（经清洗与换行策略处理后）

#### Scenario: 从文件渲染并自动分页
- **WHEN** 用户指定 `--in <file>` 读取文本
- **AND** 开启分页（默认开启或由参数显式开启）
- **THEN** 系统输出多张图片
- **AND** 输出文件名按页序号递增（例如 `out-001.png`, `out-002.png`）
- **AND** 页与页之间文本顺序与原文本一致

### Requirement: 输入源与编码
系统 SHALL 支持以下输入源：
- `stdin`：支持管道/重定向。
- `--in <file>`：从指定文件读取文本内容。

系统 SHOULD 以 UTF-8 读取与处理文本；遇到不可解码内容时 SHALL 以可读性优先的方式降级（例如替换为占位符而非崩溃）。

### Requirement: 文本清洗与规范化
系统 SHALL 在渲染前对文本做确定性清洗与规范化：
- 统一换行：处理 `\r\n`、`\n`。
- 控制字符清理：移除不可见控制字符（保留必要的 `\n` 与 `\t` 的可配置展开策略）。
- `\r`（回车覆盖行）策略：SHALL 提供一种确定性策略以提高可读性（V1 至少支持一种默认策略，如“将 `\r` 视作换行”或“保留覆盖后的最后文本”）。

### Requirement: 换行、宽度与分页
系统 SHALL 支持按统一规格进行排版：
- **宽度**：SHALL 提供一种默认宽度策略（列宽或像素宽二选一）以保证输出一致；并允许用户通过参数覆盖。
- **自动换行**：在超过宽度限制时按字符或单词边界进行换行（具体规则以实现为准，但输出必须确定性）。
- **分页**：在超过页面高度（行数或像素高度）时自动分页；每页最多容纳固定数量的渲染行；分页后输出多张图片。

### Requirement: 输出、格式与命名
系统 SHALL 支持输出为 PNG 或 JPG（默认 PNG）：
- `--out <path>`：指定输出文件路径或输出前缀。
- 多页输出时：若 `--out` 为 `name.png`，系统 SHALL 输出为 `name-001.png`, `name-002.png`；扩展名随输出格式变化。
- 输出目录不存在时：系统 SHOULD 给出清晰错误并以非 0 退出码结束。

### Requirement: 主题与字体
系统 SHALL 使用等宽字体渲染，并提供至少一个默认主题：
- `--theme <name>`：切换主题（V1 最少 `terminal` / `paper`）。
- 字体一致性：系统 SHOULD 打包并默认使用一份开源等宽字体以减少跨平台差异；若不可用，SHALL 回退到系统等宽字体。

### Requirement: 退出码与错误信息
系统 SHALL 对常见错误提供可理解的错误信息并返回非 0 退出码：
- 输入文件不存在/不可读
- 输出路径不可写
- 参数非法（如未知主题、无效格式）

## MODIFIED Requirements
（无）

## REMOVED Requirements
### Requirement: ANSI 样式还原（V1 不支持）
**Reason**: V1 聚焦“可读、可复现”的图片输出，避免引入复杂样式解析与跨平台差异。
**Migration**: 后续版本如需支持可新增 ANSI 解析与主题扩展；V1 输出将忽略颜色/粗体/下划线等样式。

### Requirement: 真实窗口截图（V1 不支持）
**Reason**: 真实截屏依赖 GUI 与终端窗口状态，无法覆盖 sandbox 等场景。
**Migration**: 不提供；V2 可选引入“HTML 伪终端 + 截图引擎”的增强路线，但不影响 V1 Render-only 能力。
